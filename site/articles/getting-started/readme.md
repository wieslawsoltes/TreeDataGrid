# Getting Started

This section is for first-time adopters and for maintainers returning to TreeDataGrid after the API surface has grown. It explains what the package ships, how the flat and hierarchical source models differ, and which guides to read after the first grid renders successfully.

## Recommended path

1. Read [Overview](overview/) to understand the control model and the main decision points.
2. Read [Installation](installation/) to set up the package, themes, and project prerequisites.
3. Follow [Quickstart: Flat](quickstart-flat/) when your data is already a flat list.
4. Follow [Quickstart: Hierarchical](quickstart-hierarchical/) when your data has parent-child expansion.

## After the first successful grid

- Continue with [Concepts](../concepts/) if you want the mental model for rows, cells, indexing, and selection.
- Continue with [Guides](../guides/) if you want scenario-driven recipes for editing, sorting, styling, and events.
- Continue with [XAML Usage](../xaml/) if you need theme resources, templates, and customization hooks.
